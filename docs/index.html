02 -- How It Works

Every action.
Mathematically verified.

BATS sits between your AI agent and your infrastructure. Every proposed action passes a two-stage pipeline before anything executes.

Stage 01
Agent Proposal

Any AI agent -- Claude Code, Cursor, AutoGen -- proposes an action via REST to the BATS leader node over mTLS.

-->
Stage 02
Heuristic Safety Gate

An LLM evaluates the action intent with a confidence score. Dangerous patterns blocked immediately. No consensus needed.

368us
Stage 03
PBFT Consensus

Safe writes are broadcast to all nodes. 2f+1 cryptographic votes required. One compromised node cannot approve alone.

6.5ms
Stage 04
Hash-Chained WAL

Every decision -- blocked or approved -- is recorded with SHA-256(PrevHash + Data). Tamper-evident. SOC2 ready.

Immutable
